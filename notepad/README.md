# 프로젝트 : Note Pad

# 프로젝트 개요 : 
사용자가 텍스트를 기록하고 관리할 수 있는 직관적인 웹 메모장 서비스입니다
복잡한 기능보다는 '기록'이라는 본질에 집중하여 깔끔한 UI와 빠른 데이터 반영을 통해 사용자에게 최적화된 메모 경험을 제공합니다

# 스킬 스택 :
Frontend: HTML5, SCSS, JavaScript, jQuery
Backend: Node.js, Express
Template Engine: Pug
Tools: npm, simplehttpserver

# 스킬 선택 이유 :
SCSS: Mixin과 변수를 활용하여 코드의 재사용성을 높이고 복잡한 스타일 구조를 체계적으로 관리하기 위해 선택했습니다
jQuery: 직관적인 DOM 조작과 이벤트 바인딩을 통해 프론트엔드 기능을 빠르게 구현하고 유지보수하기 위해 사용했습니다
Pug: HTML 구조를 간결하게 작성하고 데이터 바인딩을 효율적으로 처리할 수 있는 템플릿 엔진이기에 도입했습니다
Node.js: 가벼운 서버 환경을 구축하고 데이터를 안정적으로 관리하기 위해 선택했습니다

# 아키텍처 :
Client-Side Rendering: jQuery를 이용한 동적 UI 업데이트
Server-Side Rendering (Partial): Pug를 통한 초기 레이아웃 렌더링

# 프로젝트 구조 :
src/pug: 화면 구조를 정의하는 Pug 파일 위치
src/scss: 스타일 가이드를 담은 SCSS 파일 위치
build/: 컴파일된 정적 리소스(HTML, CSS, JS) 저장소

# 핵심 트러블 슈팅 
문제: 노트 편집 시 데이터 충돌로 인해 작성 중인 내용이 유실되는 현상 발생
원인: 여러 브라우저에서 동시 접근 시 마지막에 저장된 데이터가 이전 데이터를 덮어쓰는 구조적 한계
해결: 데이터 충돌 방지를 위한 실시간 동기화 로직을 보완하고 데이터 유실 방지를 위한 자동 저장(Auto-save) 및 버전 관리 개념을 도입하여 안정성을 강화했습니다
배운 점: 데이터의 무결성을 유지하는 것이 사용자 경험에 얼마나 직접적인 영향을 미치는지 깨달았으며 서버-클라이언트 간의 정교한 동기화 프로세스의 중요성을 학습했습니다   

# 성능 개선 수치 :
기존 시스템 대비 데이터 로딩 속도 및 응답 속도 최적화
가벼운 애플리케이션 구조로 재구성하여 초기 렌더링 시간 단축
실시간 동기화 도입 이후 데이터 충돌 보고율 0% 달성 목표
