# 프로젝트 : 주민등록번호 유효성 검사 도구

# 프로젝트 개요 :
대한민국의 주민등록번호 체계에 맞춘 실시간 유효성 검증 및 분석 서비스입니다<br>복잡한 체크섬(Checksum) 로직을 클라이언트 측에서 즉각적으로 처리하며 사용자 편의를 극대화한 UI/UX와 데이터 보안(Privacy First)을 최우선으로 설계되었습니다

# 스킬 스택 :
- Frontend: HTML5, SCSS (CSS3)
- Script: JavaScript (Vanilla JS)
- Library: jQuery (DOM manipulation 보조)
- Design: Responsive Web Design (Media Queries)

# 스킬 선택 이유 :
- Vanilla JavaScript: 외부 라이브러리 의존성을 최소화하여 가볍고 빠른 실행 속도를 보장하며 브라우저의 기본 성능을 최대한 활용했습니다
- SCSS: 변수와 Mixin을 활용하여 복잡한 스타일 구조를 모듈화하고 유지보수가 용이한 반응형 디자인을 구현했습니다
- Client-side Processing: 민감한 정보인 주민등록번호를 서버로 전송하지 않고 브라우저 내에서 즉시 처리함으로써 보안성을 극대화했습니다

# 아키텍처 :
- Separation of Concerns (관심사 분리)
- Structure (HTML): 시맨틱 태그를 사용한 웹 표준 구조 설계
- Presentation (SCSS): 레이아웃과 스타일을 분리하여 테마 변경 및 반응형 대응 용이
- Behavior (JS): 비즈니스 로직(유효성 검사)과 이벤트 핸들링을 독립적으로 관리

# 프로젝트 구조 :
- src/html: 화면 구조를 정의하는 html 파일 위치
- src/scss: 스타일 가이드를 담은 SCSS 파일 위치
- src/build/: 컴파일된 정적 리소스(CSS) 저장소
- build/css: 컴파일된 정적 리소스(CSS) 저장소
- build/js: 컴파일된 정적 리소스(JS) 저장소

# 핵심 트러블 슈팅 
1. 문제: 입력 시 비숫자(문자, 특수문자) 혼입 및 입력 흐름 단절<br>
원인: 사용자가 실수로 하이픈(-)이나 문자를 입력할 경우 유효성 검사 로직이 깨지며 앞자리 입력 후 뒷자리로 수동 이동해야 하는 번거로움 발생<br>
해결: Regex(/[^0-9]/g)를 활용한 실시간 필터링 적용 (숫자 이외 즉시 제거)<br>
앞 6자리 입력 완료 시 focus() 메서드를 통해 자동으로 뒷자리 입력창 이동 구현<br>
배운 점: 아주 작은 UX 자동화가 사용자의 피로도를 획기적으로 낮출 수 있음을 체감했습니다<br>
<br>

2. 문제: 보안에 민감한 데이터 처리 방식<br>
원인: 주민등록번호는 개인정보 중 가장 민감한 데이터로 서버 통신 시 가로채기(Sniffing) 위험이 존재<br>
해결: 폼의 onsubmit="return false;" 처리 및 모든 로직을 클라이언트 내 로컬 변수에서 처리 후 소멸되도록 설계<br>
배운 점: 보안은 기술적인 암호화뿐만 아니라 데이터의 이동 경로를 최소화하는 설계 단계부터 시작된다는 것을 배웠습니다<br>

# 성능 개선 수치 :
- 검증 속도: 로컬 연산 방식을 통해 서버 통신 대비 응답 시간 99% 단축 (0ms에 가까운 즉각 반응)
- 리소스 최적화: Vanilla JS 및 압축된 CSS 사용으로 총 에셋 크기 50KB 미만 유지
- 접근성 점수: Lighthouse 기준 모바일/데스크톱 모두 반응형 완벽 대응
